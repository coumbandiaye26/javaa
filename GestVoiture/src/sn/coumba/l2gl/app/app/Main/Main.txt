REPONSE 1/5

A — Tests (répondent VRAI/FAUX)
1.“Véhicule disponible ?”
Ce comportement permet de vérifier si un véhicule peut être utilisé ou loué à un instant donné.
Il retourne VRAI si le véhicule est libre, sinon FAUX.

2.“Véhicule en panne ?”
Ce comportement sert à savoir si le véhicule est actuellement en panne.
Il retourne VRAI si le véhicule est en panne, sinon FAUX.

3.“Kilométrage > seuil ?”
Ce comportement vérifie si le kilométrage du véhicule dépasse une valeur donnée.
Il retourne VRAI si le kilométrage est supérieur au seuil, sinon FAUX.

4.“Véhicule à réviser : (km > seuil) OU (année < seuilAnnée)”
Ce comportement permet de déterminer si un véhicule doit partir en révision.
Il retourne VRAI si le kilométrage dépasse un seuil ou si l’année du véhicule est trop ancienne.

5.“Conducteur autorisé : permis correspond à un format donné (ex: commence par 'B')”
Ce comportement vérifie si le permis du conducteur respecte une condition donnée.
Par exemple, il peut vérifier si le permis commence par "B".
Il retourne VRAI si le conducteur est autorisé, sinon FAUX.


B — Transformations (retournent une valeur)
6. “Résumé véhicule” (String)
Ce comportement produit une chaîne de caractères qui résume les informations importantes du véhicule.
Par exemple : immatriculation, marque, kilométrage, année, état.

7. “Extraire immatriculation” (String)
Ce comportement retourne uniquement l’immatriculation du véhicule sous forme de chaines.
Il permet d’obtenir rapidement son identifiant principal.

8. “Calculer âge du véhicule” (int)
Ce comportement calcule l’âge du véhicule à partir de son année de mise en circulation.
Il retourne un entier représentant le nombre d’années.

9. “Coût total d’un entretien” (int) (ex: coût + taxe fixe)
Ce comportement calcule le montant final d’un entretien.
Par exemple : coût de base + taxe fixe.
Il retourne un entier correspondant au coût total.


C — Actions (ne retournent rien)
10. “Marquer véhicule en révision”
Ce comportement modifie l’état du véhicule pour indiquer qu’il est actuellement en révision.
Il agit directement sur l’objet sans renvoyer de valeur.

11. “Augmenter kilométrage d’un véhicule de X”
Ce comportement ajoute une certaine valeur X au kilométrage actuel du véhicule.
Il sert à mettre à jour l’état réel du véhicule après un trajet.

12. “Terminer une location (mettre dateFin)”
Ce comportement permet de clôturer une location en enregistrant la date de fin.
Il modifie l’objet location pour indiquer qu’elle est terminée.


D — Comparaison / Tri (ordre entre 2 éléments)
13. “Comparer deux véhicules par kilométrage (ordre croissant)”
Ce comportement permet de comparer deux véhicules selon leur kilométrage.
Le véhicule avec le plus petit kilométrage doit apparaître en premier.

14. “Comparer deux véhicules par immatriculation (ordre alphabétique)”
Ce comportement permet de comparer deux véhicules selon leur immatriculation.
Le tri se fait dans l’ordre alphabétique croissant.
